# 상세페이지 캡쳐 웹앱 — 배포 가이드 (개발 몰라도 따라할 수 있게)

## 이게 뭐냐면

브라우저에서 URL 입력 → 캡쳐하기 버튼 → 고화질 이미지 다운로드,
이렇게 되는 웹사이트예요. 완성되면 링크 하나로 지인들한테 공유하면 끝.

## 전체 흐름 (5단계)

1. GitHub에 이 코드 올리기 (코드 저장소)
2. Render 가입하기 (무료 서버 호스팅 사이트)
3. Render에서 "이 GitHub 코드로 서버 띄워줘" 연결하기
4. 몇 분 기다리면 링크 생성됨
5. 그 링크를 지인들한테 공유

**Antigravity를 쓰면 1번, 3번 과정의 실제 명령어 입력을 AI가 대신 해줄 수 있어요.**
아래는 그래도 전체 흐름을 이해하기 위한 설명이에요.

---

## 1단계: GitHub에 코드 올리기

GitHub은 코드를 저장해두는 곳이에요 (네이버 카페 같은 느낌으로 "코드 저장소"라고 생각하면 됨).

1. [github.com](https://github.com) 가입 (무료)
2. 오른쪽 위 `+` 버튼 → `New repository` → 이름 아무거나 (예: `detail-capture-web`) → `Create repository`
3. Antigravity한테 "이 폴더를 방금 만든 GitHub 저장소에 올려줘"라고 시키면 대신 해줌
   (직접 하고 싶으면 Antigravity가 알려주는 `git push` 명령어를 순서대로 실행하면 됨)

## 2단계: Render 가입

1. [render.com](https://render.com) 접속 → GitHub 계정으로 가입 (가장 간단함)

## 3단계: Render에서 서버 만들기

1. Render 대시보드에서 `New +` → `Web Service` 클릭
2. 아까 만든 GitHub 저장소(`detail-capture-web`) 선택
3. 설정 화면에서:
   - **Name**: 아무 이름 (예: `my-detail-capture`)
   - **Runtime**: `Docker` 선택 (자동으로 인식되는 경우가 많음, 이 프로젝트에 Dockerfile이 이미 들어있어서 별도 설정 거의 불필요)
   - **Instance Type**: `Free` 선택 ⭐ 이게 무료 옵션
4. `Create Web Service` 클릭
5. 5~10분 정도 기다리면 (처음엔 좀 오래 걸려요) 화면에 초록불 켜지면서
   `https://my-detail-capture.onrender.com` 같은 링크가 생성됨

## 4단계: 접속 테스트

생성된 링크로 접속해서 아무 상세페이지 URL 넣고 캡쳐 버튼 눌러보기.
잘 되면 그 링크를 지인들한테 카톡으로 보내면 끝!

---

## 무료 플랜 특징 (미리 알아두면 좋은 것)

- **15분 동안 아무도 안 쓰면 서버가 잠들어요.** 그 상태에서 첫 접속하면
  30초~1분 정도 "깨어나는 시간"이 걸려요. 지인 몇 명이 하루 몇 번 쓰는
  규모면 크게 불편하지 않을 거예요.
- **메모리가 넉넉하지 않아서**, 너무 무거운 페이지나 해상도를 너무 높게
  잡으면(scale 3~4) 실패할 수 있어요. 그래서 웹앱 버전은 기본 배율을
  2로 낮추고, 최대 2.5로 제한해뒀어요. (그래도 화면 캡쳐치고는 충분히 선명함)
- **여러 명이 동시에 누르면** 서버가 순서대로 하나씩 처리해요
  (동시에 여러 개 돌리면 무료 서버가 못 버텨서 아예 이렇게 설계함).
  두 번째 사람은 첫 번째 캡쳐 끝날 때까지 살짝 기다리게 됨.

## 나중에 사용자가 늘어나서 느려지면

- Render 유료 플랜으로 올리기 (월 7천원대부터, 항상 켜져있고 메모리도 더 큼)
- 또는 그냥 이 상태로 계속 쓰다가 불편해지면 그때 업그레이드 고려

## 로컬에서 먼저 테스트해보고 싶다면 (선택사항)

배포 전에 내 컴퓨터에서 잘 되는지 먼저 확인하고 싶으면:

```
npm install
npx playwright install chromium
node server.js
```

그다음 브라우저에서 `http://localhost:3000` 접속하면 됨.
